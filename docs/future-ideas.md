# Future Ideas

## Goal of This Document
Capture credible post-MVP expansions while preserving the local-first identity.

## Intelligence and Theme Modeling
- **Deeper theme normalization:** move beyond rule-based mapping to richer local embedding-based clustering.
- **Richer local AI:** run compact on-device models for better context-sensitive theme extraction.
- **Theme hierarchy:** support parent/child relationships (e.g., "work" -> "career uncertainty").
- **User-guided ontology:** let users define personal theme categories over time.

## Reflection and Insight Features
- **Journaling timeline view:** chronological stream linking voice notes, transcripts, and graph updates.
- **Trend analysis:** identify rising, fading, and cyclical themes over weeks/months.
- **Pattern summaries:** periodic local summaries like "most frequent theme this week".
- **Reflective session mode:** guided multi-step prompts around selected themes.

## Symbolic and Spatial Evolution
- **Symbolic/archetypal layers:** optional interpretation overlays selected by user preference.
- **Map-like countries:** evolve from isolated nodes to themed regions on a map-like surface.
- **Cluster territories:** detect and visualize neighborhoods of related themes.
- **Temporal graph playback:** scrub through time to watch graph evolution.

## Usability and Exploration
- **Search and history:** quickly find past voice notes, transcripts, and theme nodes.
- **Manual theme editing/merging:** allow users to merge duplicates, rename themes, or split broad themes.
- **Theme pinning:** keep selected nodes highlighted for focused reflection sessions.
- **Custom visual modes:** switch between 3D sphere, 2D map, and list-based representations.

## Portability and Trust
- **Optional export/import:** portable encrypted backups for user ownership.
- **Optional encrypted sync:** user-controlled multi-device sync with strong privacy defaults.
- **Data retention controls:** configurable deletion/archiving policies for audio and transcript history.
- **Local privacy diagnostics:** explain where data lives and how it is protected.

## Accessibility and Global Reach
- **Multi-language support:** locale-aware transcription and theme normalization.
- **Accessibility modes:** high contrast graph theme, reduced motion, screen reader-first summaries.
- **Low-resource device mode:** simplified rendering for performance-constrained devices.

## Product Evolution Strategy
- Keep future enhancements modular so core MVP loop remains lightweight.
- Maintain a strict offline-first baseline; add network features only as explicit opt-in layers.
- Prioritize features that increase reflective clarity over feature volume.
